# 🧱 Software Principles

**Skill de especialista em princípios de software** — SOLID, OO e Pragmáticos, com foco em identificar violações e recomendar correções simples.

## Para que serve?

Esta skill instrui o agente a analisar código sob a ótica dos princípios fundamentais de design de software, identificando violações e recomendando correções com foco em **simplicidade** e **pragmatismo**.

> "Princípios são guias, não regras absolutas. O contexto sempre vence a teoria."

## Princípios Cobertos

### SOLID
| Princípio | Essência |
|-----------|----------|
| **SRP** — Single Responsibility | Uma classe, um motivo para mudar |
| **OCP** — Open/Closed | Aberto para extensão, fechado para modificação |
| **LSP** — Liskov Substitution | Subtipos substituíveis sem quebrar o programa |
| **ISP** — Interface Segregation | Interfaces focadas, sem métodos desnecessários |
| **DIP** — Dependency Inversion | Dependa de abstrações, não de concretos |

### Design OO
- Lei de Demeter (LoD)
- Tell, Don't Ask
- Composition over Inheritance
- Program to an Interface
- Encapsulate What Varies
- Principle of Least Astonishment (POLA)

### Pragmáticos
- DRY, KISS, YAGNI, Boy Scout Rule, Fail Fast

## Quando usar?

- Ao revisar código para identificar violações a SOLID ou princípios OO
- Para obter recomendações de refatoração com exemplos de código
- Como guia de boas práticas em design de software
