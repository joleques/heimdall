# 🎯 Design Patterns Specialist

**Skill de especialista pragmático em Design Patterns (GoF)** com foco em simplicidade primeiro.

## Para que serve?

Esta skill instrui o agente a analisar e recomendar o uso de Design Patterns de forma **pragmática**, sabendo quando usar e — mais importante — **quando NÃO usar** patterns, evitando over-engineering.

## Filosofia

> "O melhor código é aquele que não precisa de um pattern. Patterns são ferramentas, não objetivos."

A skill segue os princípios YAGNI, KISS e a Regra dos 3 (veja o problema acontecer 3 vezes antes de abstrair).

## Patterns Cobertos

| Categoria | Patterns |
|-----------|----------|
| **Creational** | Factory Method, Abstract Factory, Builder, Singleton, Prototype |
| **Structural** | Adapter, Bridge, Composite, Decorator, Facade, Flyweight, Proxy |
| **Behavioral** | Chain of Responsibility, Command, Iterator, Mediator, Memento, Observer, State, Strategy, Template Method, Visitor |

Para cada pattern, a skill define **quando usar** e **quando evitar**.

## Quando usar?

- Ao revisar código para identificar patterns desnecessários
- Antes de aplicar um pattern, para validar se é realmente necessário
- Para obter recomendações de simplificação
