# ✍️ Engineering Writer

**Skill de escrita de artigos técnicos** sobre arquitetura e engenharia de software — transforma experiência prática em conteúdo claro, provocativo e útil para engenheiros.

## Para que serve?

Esta skill instrui o agente a escrever artigos técnicos com **profundidade**, **tom humilde e levemente irônico**, e fundamentados em **experiência prática**. O objetivo é produzir conteúdo que ensine algo útil e provoque reflexão — sem hype, sem buzzwords, sem "verdade absoluta".

> "Escreva para compartilhar experiência, não para parecer mais inteligente."

## Perfil do Autor

O agente escreve como um **arquiteto de software experiente**, com domínio em:

| Área | Experiências |
|------|-------------|
| **Design & Arquitetura** | DDD, Clean Architecture, Design Patterns, arquitetura evolutiva |
| **Infraestrutura** | Microsserviços, Docker, Kubernetes, CI/CD, sistemas distribuídos |
| **Práticas** | Refatoração contínua, testes automatizados, observabilidade |

## Estrutura dos Artigos

Cada artigo segue 5 seções:

1. **Problema ou Provocação Inicial** — abre com situação real
2. **Contexto Técnico** — relaciona com práticas reconhecidas
3. **Experiência Prática** — observações do mundo real
4. **Explicação ou Análise** — trade-offs e abordagens possíveis
5. **Conclusão** — reflexão prática

## Formatos Suportados

| Formato | Tamanho |
|---------|---------|
| Post LinkedIn | 150–300 palavras |
| Artigo médio | 600–1000 palavras |
| Artigo aprofundado | 1000–1800 palavras |

## Quando usar?

- Para escrever artigos técnicos sobre arquitetura de software
- Para criar posts de blog ou LinkedIn
- Para transformar experiências de engenharia em conteúdo
- Geralmente usada via workflow [`/write-article`](../../workflows/write-article.md)
